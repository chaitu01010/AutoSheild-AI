About the Project

Securing the Future of Smart Mobility

A hackathon-built defense layer that lives at the edge of the vehicle — not in the cloud.

AutoShield AI is an edge-deployed intrusion detection and response system purpose-built for connected and autonomous vehicles. It listens to CAN bus traffic in real time, flags anomalies with lightweight machine learning models, and triggers a response — all without a round trip to the cloud.

The Problem

Connected Vehicles Are Wide Open

Every connected ECU is a new door. Today, most of those doors have no lock.

CAN Bus Vulnerability

The Controller Area Network has no built-in authentication, letting any connected node broadcast messages the vehicle will trust.

Spoofing

Attackers impersonate legitimate ECUs, injecting fake sensor or command data to mislead the vehicle's decision-making.

Replay Attack

Previously captured CAN messages are re-transmitted to trigger unintended vehicle actions, bypassing normal validation.

Denial of Service

Flooding the bus with high-priority frames starves critical ECUs of bandwidth, degrading or halting real-time control.

Unauthorized Commands

Malicious payloads can trigger braking, steering, or acceleration commands without driver consent or system authorization.

Cloud Detection Delay

Cloud-based security analysis introduces latency and connectivity dependence — unacceptable for millisecond-critical threats.
